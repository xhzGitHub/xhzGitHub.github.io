    # 亮点

    • Performance: 性能比 vue2 快 1.2~2 倍
    • Tree Shaking support: 按需编译, 体积比 vue2 更小
    • Composition API: 组合 API（类似 React Hooks）
    • Better TS support: 更好的 TS 支持
    • Custom Renderer API: 暴露了自定义渲染 API
    • Fragment, Teleport, Suspense: 更先进的组件

    # Diff 算法优化

    用以下 DOM 作为简单的例子：

    <div>
    <p>我是段落</p>
    <span>{{ msg }}</span>
</div>

    以下是vue 2 与 3 中不同的 diff 算法区别:

    • Vue2 全量对比，层级由浅至深，逐层对比，全量比较，将发生数据改动的标签做修改。以上例子中对比了 div，p，span 都做了对比

    • Vue3 创建虚拟 dom 时为以后可能发生改变的 dom 元素添加静态标记，在 diff 时只对比带静态标记的节点。以上例子中在创建 dom 时只为 span 标签创建静态标记，diff 时只对 span 作对比。

    通过个稍微复杂一些的例子讲解，以下链接为将 vue dom 转换为 vue3 创建 vnode 源码的过程： （链接 https://vue-next-template-explorer.netlify.app/）

    可以看到，静态节点与动态节点在创建 vnode 时的区别，vue3 通过 _toDisplayString 来动态节点做静态标 （PatchFlag）记用于 diff

    PatchFlag 也是有类型表的，比如 1 代表动态文本节点，128代表带有 key 属性的 fragment 或部分子字节有 key

    _createVNode("span", { class: "middle" }, _toDisplayString(tag), 1

    # hoistStatic 静态提升

    • Vue2 中无论元素是否参与更新，每次都会被重新创建与渲染
    • Vue3 中对于不参与更新的元素，会做静态提升，只会被创建一次，在渲染时直接复用，牺牲内存提升效率！

    如下图所示, 每次调用 render 方法都需要重新创建节点 createVNode

    但加入了静态提升后，将变量提升到全局，每次 render 时调用即可，不需要重复创建

    # setup 本质

    [本质]：其实 Composition API 的本质就是将 setup 函数 return 出来的数据与函数分别注入到 Optional API 中的 data 与 methods 中。

    [时机]：setup 是在 beforeCreate 之前执行的，因为 beforeCreate 时刚刚 new Vue() 初始化 vue 实例，初始化生命周期与一些方法，而此时 setup 也需要将返回的数据注入到初始化状态中。并且此时 this 为 undefined。

    为什么此时 this = undefined ？

    因为 babel 转译时使用了严格模式（“use strict”），严格模式下无法意外创建全局变量，所以在全局下调用函数时，函数中的 this 不再指向 window 而是 undefined。

    # 递归监听

    Vue3 对引用类型做了深层递归，也就是每一层对象元素都转换为 Proxy 对象，当对象深度很多时，会非常影响性能。

    shallowReactive [new API] shallowRef [new API]

    以上2个 API 分别会让对象只被监听第一层，不做深层递归监听。
