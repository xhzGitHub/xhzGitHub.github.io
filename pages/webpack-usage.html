                      # 安装(install)

                      npm i -D webpack webpack-cli

                      # 简单使用(run)

                      创建如下文件夹目录：

                      project/
--| src/
-----| index.js       入口文件
--| package.json

                      在index.js 文件中随便写点什么，作为打包的"入口文件"。

                      输入指令

                      npx webpack

                      webpack自动将 index.js 打包成功！

                      可见，直接运行 webpack 命令默认将 src 中的 index.js 作为入口文件进行打包。

                      (为什么要用 npx, 因为 webpack 不是全局安装，npx 指令是去 node module 中寻找相应的 webpack 进行执行)

                      # 适当配置(config)

                      我们可以通过配置文件 webpack.config.js 来运行 webpack。

                      目录如下：

                      project/
--| src/
-----| index.js        入口文件
--| package.json
--| webpack.config.js   打包📦配置文件

                      在配置文件中进行设置：

                      const path = require("path");

module.exports = {
  mode: "development",      // 开发模式
  entry: "./src/index.js",  // 入口文件
  output: {                 // 输入路径
    path: path.resolve(__dirname, "dist"),
    filename: "[name].js"
  },
}

                      (path.resolve 请查看文档)

                      配置中 __dirname 为当前运行文件的目录，path.resolve 将其拼接为绝对路径 /Usr/xiaohongze/practise/webpack_day1/dist

                      # loader

                      webpack 本身只能打包📦 js 文件，其它类型的文件并不识别(比如 css, js 等)。webpack 可以借助 loader 可以解决这个问题。

                      # file-loader

                      文件解析。

                      const path = require("path");

module.exports = {
  // ...
  module: {
    rules: [
      {
        test: /\.png$/,
        use: {
          loader: "file-loader"
        },
      }
    ]
  },
}

                      file loader 会向指定文件夹中的文件复制一份，并修改其名称到输出文件夹中。

                      # css-loader

                      css 解析。解析顺序为自后往前，自右往左。

                      module: {
    rules: [
      // ...
      {
        test: /\.scss$/,
        use: [
          "style-loader", // 将 JS 字符串生成为 style 节点
          "css-loader",   // 将 css 转化为 commonJS 模块
          "sass-loader",  // 将 sass 编译成 css, 默认使用 node-sass
        ]
      }
    ]
},

                      解析前后的文件长这样：

                      解析前：

                      .app {
  width: 100px;
  height: 100px;

  .img {
    width: 100%;
    height: 100%;
  }
}

                      解析后：

                      /***/ (function(module, exports, __webpack_require__) {

eval("// Imports\nvar ___CSS_LOADER_API_IMPORT___ = __webpack_require__(/*! ../../node_modules/_css-loader@3.4.0@css-loader/dist/runtime/api.js */ \"./node_modules/_css-loader@3.4.0@css-loader/dist/runtime/api.js\");\nexports = ___CSS_LOADER_API_IMPORT___(false);\n// Module\nexports.push([module.i, \".app {\\n  width: 100px;\\n  height: 100px; }\\n  .app .img {\\n    width: 100%;\\n    height: 100%; }\\n\", \"\"]);\n// Exports\nmodule.exports = exports;\n\n\n//# sourceURL=webpack:///./src/css/index.scss?./node_modules/_css-loader@3.4.0@css-loader/dist/cjs.js!./node_modules/_sass-loader@8.0.0@sass-loader/dist/cjs.js");

/***/ }),

                      style-loader 能将解析的 style 样式在 html 文件中动态生成 style 标签。

                      # postcss-loader

                      为 css 样式自动添加前缀。

                      # url-loader

                      能做 file-loader 的所有事，但也有不同：

                      • 输出的图片文件默认不会直接输出到 outputPath 中，而是将图片转化为 base64 格式到 main.js 中。 可以设置 limit 属性的阈值，当超过阈值时，不会转成 base64 格式。

                      # plugins

                      plugins 很关键! ，插件目的在于解决 loader 无法实现的其他事。

                      在每次使用 plugins 时，都需要下载对应插件，然后做以下事情：

                      • 引入：const XxxPlugin = require('xxx-webpack-plugin')
                      • 实例化并引用，plugins: [ new XxxPlugin() ]

                      # html-webpack-plugin

                      拿 htmlWebpackPlugin 举个例子，该插件的作用在于将目标 html 文件复制到 output 文件夹中，并自动将 main.js 以 <script> 标签嵌入：

                      新增 html 文件于 /src/中，目录如下：

                      project/
--| src/
-----| index.html      模版 html 文件
-----| index.js        入口文件
--| package.json
--| webpack.config.js   打包📦配置文件

                      // webpack.config.js

                      const HtmlWebpackPlugin = require('html-webpack-plugin');

module.exports = {
    // ...
    plugins: [
      new HtmlWebpackPlugin({
        template: "./src/index.html"
      })
    ]
}

                      # clean-webpack-plugin

                      作用：打包📦前删除 output 文件夹。

                      const { CleanWebpackPlugin } = require('clean-webpack-plugin');

module.exports = {
    // ...
    plugins: [
      new CleanWebpackPlugin(),
    ]
}

                      # mini-css-extract-plugin

                      之前都是通过 style-loader 将 css 转为样式并嵌入 <style> 标签中，代替 style-loader， mini-css-extract-plugin 可以将 css 文件单独提出放到 css 文件中。

                      # source-map

                      源代码与打包后的代码的映射关系。

                      他有什么用呢？

                      在平时写代码写出 bug 的时候，我在 index.js中 写了这样一行代码：

                      consoe.log('xxx')

                      你会在网页上看到这样的结果：

                      然后你再点进这个错误的时候会显示：

                      这里的 main.js 其实就是 webpack 打包生成的文件，点进去的代码则是打包后的 main.js 中的代码。

                      source-map 的作用是为了将源代码与打包后的代码对应起来，即使报错了也不会直接告诉你打包后的错误位置，而是告诉你打包前的，做如下配置：

                      module.exports = {
    // ...
    devtools: "source-map"
}

                      报错如下： 可见，最后将源代码出错的位置报了出来。

                      在打包后的文件中，生成了 main.js.map 映射文件。

                      一般项目的推荐配置：

                      devtool: "cheap-module-eval-source-map" // 开发环境
devtool: "cheap-module-source-map"  // 生产环境

                      (cheap 配置可以省略行信息，加快打包速度)

                      source-map 的更多配置请看 链接

                      # webpack-dev-server

                      作用：能够在开发时实时监听改变并实时刷新。

                      启动服务后的 dist 文件夹没有了，这是因为 devServer 将打包后的模块放到内存中，从而提升速度。

                      在 webpack.config.js 中配置：

                      module.exports = {
  // ...
  devServer: {
    contentBase: "./.dist",
    open: true,
    port: 8080,
    proxy: {
      "/api": {
        target: "http://localhost:9092"
      }
    },
  },
}

                      其中 proxy 代表跨域。

                      # Hot Module Replacement 热模块替换(HMR)

                      在使用了 webpack dev server 后，每当修改文件时，整个网页会实时刷新，但痛点是无法只更新修改的部分，未修改的部分不更新，HRM 可以解决。

                      但是，只对 css 有效，对 js 无效

                      配置 webpack.config.js:

                      devServer: {
    contentBase: "./dist",
    open: true,
    hot:true, //即便HMR不生效，浏览器也不⾃动刷新，就开启hotOnly
    hotOnly:true,
},

plugins: [
    new HtmlWebpackPlugin({
        template: "./src/index.html"
    }), // 这个插件也需要
    new webpack.HotModuleReplacementPlugin(), // 热加载插件
],

                      # Babel

                      作用：将 es6 语法转换为 es5。

                      配置：

                      rules: [
  {
    test: /\.js$/,
    include: path.resolve(__dirname, './src'),
    loader: "babel-loader",
    options: {
      presets: ["@babel/preset-env"]
    }
  }
]

                      其中，@babel/preset-env 是语法转换的核心。babel-loader 是 webpack 与 babel 的通信桥梁梁，不不会做把 es6 转成 es5 的⼯工作，这部分⼯工作需要⽤到 @babel/preset-env 来做。

                      # @babel/polyfill

                      @babel/polyfill 能够将所有 es6 语法引进来，挂在 window 全局变量上，用于将一些 es6 的语法能够让浏览器看得懂，比如 Promise 等。

                      但正是因为污染了 window 全局变量，所以不适用于一些场景，所以不推荐使用。

                      # .babelrc

                      babel 的配置文件，当 webpack 中没有 options 时，就会自动读取 .babelrc 文件中的配置。

                      # React 解析

                      只需2步：

                      • 写一些 React 代码;
                      • 配置 webpack 与 React;

                      (1) 在 index.js 中写 React 组件，将组件挂载在 index.html 中：

                      import React, { Component } from 'react';
import ReactDOM from 'react-dom';

class index extends Component {
  render() {
    return (
      <div>
        hello
      </div>
    )
  }
}

ReactDOM.render(<Index />, document.getElementById('app'));

                      (前提安装 react, react-dom)

                      (2) 安装 @babel/preset-react

                      配置 .babelrc:

                      {
  "presets": ["@babel/preset-react"]
}

                      大功告成！

                      # Vue 解析

                      需要下载 vue-loader 与 vue-template-compiler，前者用于将 vue 代码解析为 js，后者用于解析 vue 模板。

                      (1) 写一个 vue 组件 // App.vue

                      <template>
  <div>hello</div>
</template>

<script>
export default {}
</script>

                      (2) 将 vue 组件挂载到 index.html 下： // index.js

                      import App from './app.vue';
import Vue from 'vue';

new Vue({
  el: '#app',
  render: h => h(App)
});

                      (3) 配置 webpack.config.js:

                      const VueLoaderPlugin = require('vue-loader/lib/plugin');

module.exports = {
    module: {
        rules: [
          {
            test: /\.vue$/,
            loader: "vue-loader",
          }
        ]
    },
    plugins: [
        new VueLoaderPlugin()
    ]
}

                      大功告成！

                      # 开发环境 & 生产环境

                      在打包时，开发环境(development)与生产环境(production)的配置会有不同，因此可以将开发与生产环境的配置分开。

                      在根目录下新建文件夹：

                      webpack/
--| build/
-----| webpack.base.js       公共配置
-----| webpack.dev.js        开发环境配置
-----| webpack.pro.js        生产环境配置

                      分别将开发与生产环境下的公共配置提取到 webpack.base.js 中，并且各自保持独有的配置。

                      最后需要将两种环境下的配置与公共配置进行融合。

                      安装 webpack-merge。

                      yarn add -D webpack-merge

                      分别配置三个文件：

                      // webpack.base.js

                      const baseConfig = {
    // ...公共配置
};
module.exports = baseConfig;

                      // webpack.dev.js

                      const merge = require('webpack-merge');
const baseConfig = require('./webpack.base.config');

const devConfig = {
    // ...开发环境的配置
};

module.exports = merge(baseConfig, devConfig);

                      // webpack.pro.js

                      const merge = require('webpack-merge');
const baseConfig = require('./webpack.base.config');

const proConfig = {
    // ...开发环境的配置
};

module.exports = merge(baseConfig, proConfig);

                      最后修改命令行 package.json

                      "script": {
    "dev": "webpack-dev-server --config ./build/webpack.dev.js",
    "build": "webpack --config ./build/webpack.pro.js",
}

                      # 代码分割

                      在引入一些体积非常大的第三方库时，需要将第三方库与逻辑代码分开打包，我们希望最后打包成这样：

                      <script scr="lodash.js"></script>
<script scr="main.js"></script>

                      我们只需要配置：

                      // webpack.config.js

                      module.exports = {
  // 自动代码分割
  optimization: {
    splitChunks: {
      chunks: "all",  // 默认支持异步
    }
  }
}
