          # 概述

          # Vue-Router 大致过程

          Vue-Router 的中心思想：采用 hash 模式 或 history 模式，对路由的变化进行实时监控，当路由变化时，能够渲染对应当前 url 对应的路由配置中的组件。

          Vue-Router 的核心原理是：创建一个可以储存路由信息(比如 url, query, name, component 等) 的变量，暂起名为 current，并且加以观察者模式数据劫持实时监控，当路由发生变化时，重新 set 该变量，watcher 监听到变化后执行相应的渲染函数，改变 router-view 中的视图。

          Vue-Router 初始化时：

          • 创建了 router-view 和 router-link 2个全局组件；
          • 通过传入的参数 route 配置创建了 matcher 匹配器，用于创建 routeRecord 与 路由映射表；
          • 添加 current_route 属性，并且绑定数据监听响应化；
          • 在用户点击了 router-link 或直接修改 url，通过监听事件 hashChange 来监听 url 的 hash 变化，更新浏览器历史页面栈(update history)。当 hash 变化后，执行 vueRouter 实例的 push 或 replace 方法，其中会调用 transtionTo；
          • 调用 transtionTo 的方法做路由转换，从 match map 映射表中找出对应的需要渲染的组件(component)，修改当前的 current_route；
          • current_route 发生改变，通过订阅通知(notify)，调用 vue 实例的 render 方法，渲染 route-view 组件，对应是 route 中配置的 component;

          # 自己造类似的轮子

          • 初始化
          import HashHistory from "./history/hash";
import {observer} from "./wathcer/observer";
import {Watcher} from "./wathcer/watcher"

class XRouter {
  constructor(options) {
    this.routes = options.routes; // 创建时传入的路由配置
    this.container = options.id; // router-view 的 id
    this.mode = options.mode || 'hash'; // 当前模式：分 hash、history、abstract，当前我们只实现 hash
    this.fallback = this.mode === 'history' && options.fallback !== false; // 是否采用“降级模式”
    // this.history = this.history ? new Html5History : new HashHistory;
    this.history = new HashHistory(this); // 创建一个 History 类，用于储存路由信息等;
    this.init(); // 初始化
  }

  // 在使用时路由跳转的方法，比如前进是 router.push、重定向是 router.replace，他们间接调用是 `History` 类中的方法；
  push(location) {
    this.history.push(location)
  }

  replace(location) {
    this.history.replace(location)
  }

  go (n) {
    this.history.go(n)
  }

  // init 为初始化时执行的函数，包括了对 history 实例中的 current 变量进行监控，初始化当前的 current 等
  init() {
    const history = this.history;
    observer.call(this, this.history.current);
    new Watcher(this.history.current, 'route', this.render.bind(this))
    history.push(history.getCurrentLocation());
  }

  // 用于在 current 变化(set)后的渲染函数；
  render() {
    let i
    if ((i = this.history.current) && (i = i.route) && (i = i.component)) {
      document.getElementById(this.container).innerHTML = i
    }
  }
}

export default XRouter;

          (2) 创建基类 Base，HashHistory 是他的子类，它主要的作用就是做了 HashHistory 与 Html5History 公共的事情，其中包括了创建路由映射表(macher), 路由跳转时执行的方法(transitionTo)等；

          export default class Base {
  constructor (router) {
    this.router = router
    this.current = {
      path: '/',
      query: {},
      params: {},
      name: '',
      fullPath: '/',
      route: {}
    }
  }

  /**
   * 路由转换
   * @param target 目标路径
   * @param cb 成功后的回调
   */
  transitionTo(target, cb) {
    // 通过对比传入的 routes 获取匹配到的 targetRoute 对象
    const targetRoute = match(target, this.router.routes);
    console.log('targetRoute', targetRoute);

    this.confirmTransition(targetRoute, () => {
      this.current.route = targetRoute
      this.current.name = targetRoute.name
      this.current.path = targetRoute.path
      this.current.query = targetRoute.query || getQuery()
      this.current.fullPath = getFullPath(this.current)
      cb && cb()
    })
  }

  /**
   * 确认跳转
   */
  confirmTransition (route, cb) {
    // 钩子函数执行队列
    let queue = [].concat(
      this.router.beforeEach,
      this.current.route.beforeLeave,
      route.beforeEnter,
      route.afterEnter
    )
    // 通过 step 调度执行
    let i = -1
    const step = () => {
      i ++
      if (i > queue.length) {
        cb()
      } else if (queue[i]) {
        queue[i](step)
      } else {
        step()
      }
    }
    step(i)
  }
}

function getFullPath ({ path, query = {}, hash = '' }, _stringifyQuery){
  const stringify = _stringifyQuery || stringifyQuery
  return (path || '/') + stringify(query) + hash
}

// 寻找 path 在 routeMap 中的映射关系，返回 path 对应的 route 对象
export function match(path, routeMap) {
  let match = {}
  if (typeof path === 'string' || path.name === undefined) {
    for(let route of routeMap) {
      if (route.path === path || route.path === path.path) {
        match = route
        break;
      }
    }
  } else {
    for(let route of routeMap) {
      if (route.name === path.name) {
        match = route
        if (path.query) {
          match.query = path.query
        }
        break;
      }
    }
  }
  return match;
}

          （3）创建 HashHistory 类，主要负责将 url 转化为 hash 规范，以及路由实际的跳转功能：

          import Base, { match } from './base';

export default class HashHistory extends Base {
  constructor(router) {
    super(router);
    this.ensureSlash();
  }

  ensureSlash () {
    const path = this.getCurrentLocation()
    if (path.charAt(0) === '/') {
      return true
    }
    changeUrl(path)
    return false
  }

  getCurrentLocation() {
    const href = window.location.href
    const index = href.indexOf('#')
    return index === -1 ? '' : href.slice(index + 1)
  }

  push (location) {
    const targetRoute = match(location, this.router.routes)
    this.transitionTo(targetRoute, () => {
      changeUrl(this.current.fullPath.substring(1))
    })
  }

  replaceState (location) {
    const targetRoute = match(location, this.router.routes)

    this.transitionTo(targetRoute, () => {
      changeUrl(this.current.fullPath.substring(1), true)
    })
  }
}

// 如果 url 为非 hash 类型，则将其添加 "#"
function changeUrl(path, replace) {
  const href = window.location.href
  const i = href.indexOf('#')
  const base = i >= 0 ? href.slice(0, i) : href
  if (replace) {
    window.history.replaceState({}, '', `${base}#/${path}`)
  } else {
    window.history.pushState({}, '', `${base}#/${path}`)
  }
}

          在 Vue 中，一般使用 Vue.use(xxx)来注入插件。

          export function initUse (Vue: GlobalAPI) {
  Vue.use = function (plugin: Function | Object) {
    const installedPlugins = (this._installedPlugins || (this._installedPlugins = []))
    if (installedPlugins.indexOf(plugin) > -1) {
      return this
    }

    const args = toArray(arguments, 1)
    args.unshift(this)
    if (typeof plugin.install === 'function') {
      plugin.install.apply(plugin, args)
    } else if (typeof plugin === 'function') {
      plugin.apply(null, args)
    }
    installedPlugins.push(plugin)
    return this
  }
}

          # Vue.use

          Vue.use 接受一个 plugin 参数，并且维护了一个 _installedPlugins 数组，它存储所有注册过的 plugin；接着又会判断 plugin 有没有定义 install 方法，如果有的话则调用该方法，并且该方法执行的第一个参数是 Vue；最后把 plugin 存储到 installedPlugins 中。

          可以看到 Vue 提供的插件注册机制很简单，每个插件都需要实现一个静态的 install 方法，当我们执行 Vue.use 注册插件的时候，就会执行这个 install 方法，并且在这个 install 方法的第一个参数我们可以拿到 Vue 对象，这样的好处就是作为插件的编写方不需要再额外去 import Vue 了。

          # Vue-Router 初始化

          构造函数定义了一些属性，其中:

          • this.app 表示根 Vue 实例，this.apps 保存持有 $options.router 属性的 Vue 实例;
          • this.options 保存传入的路由配置;
          • this.beforeHooks、 this.resolveHooks、this.afterHooks 表示一些钩子函数，我们之后会介绍;
          • this.matcher 表示路由匹配器，我们之后会介绍，* this.fallback 表示在浏览器不支持 history.pushState 的情况下，根据传入的 fallback 配置参数，决定是否回退到hash模式;
          • this.mode 表示路由创建的模式，this.history 表示路由历史的具体的实现实例，它是根据 this.mode 的不同实现不同，它有 History 基类，然后不同的 history 实现都是继承 History。

          路由初始化过程是在组件的初始化阶段 beforeCreate 阶段，执行 router.init() 方法，然后又会执行 router.init 方法，执行history.transtionTo(), 进而引出了 matcher 的概念，接下来我们先研究一下 matcher 的相关实现。
